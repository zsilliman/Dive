2 // CUIIRFilter.h
3 // Cornell University Game Library (CUGL)
4 //
5 // This class is represents a infinite impulse response filter. It is a
6 // general purpose filter that allows an arbitrary number of coefficients.
7 // It should only be used for 3-order or higher IIRs. In all other cases,
8 // you should use one of the more specific filters for performance reasons.
9 //
10 // This class supports vector optimizations for SSE and Neon 64. In timed
11 // simulations, these optimizations provide at least a 3-4x performance
12 // increase (and in isolated cases, much higher). These optimizations make
13 // use of the matrix precomputation outlined in "Implementation of Recursive
14 // Digital Filters into Vector SIMD DSP Architectures"
15 //
16 // https://pdfs.semanticscholar.org/d150/a3f75dc033916f14029cd9101a8ea1d050bb.pdf
17 //
18 // The algorithm in this paper performs extremely well in our tests, and even
19 // out-performs Apple's Acceleration library. However, our implementation is
20 // limited to 128-bit words as 256-bit (e.g. AVX) and higher show no significant
21 // increase in performance.
22 //
23 // For performance reasons, this class does not have a (virtualized) subclass
24 // relationship with other IIR or FIR filters. However, the signature of the
25 // the calculation and coefficient methods has been standardized so that it
26 // can support templated polymorphism.
27 //
28 // This class is NOT THREAD SAFE. This is by design, for performance reasons.
29 // External locking may be required when the filter is shared between multiple
30 // threads (such as between an audio thread and the main thread).
